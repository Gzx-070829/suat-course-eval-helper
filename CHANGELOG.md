# Changelog

本项目的显著变化记录于此。

## [1.1.0] - 2026-07-06

- 所有 textarea 写入改为“写入、等待、回读验证”，不一致时最多重试 2 次。
- Element UI 评分下拉框点击选项后必须精确回读“非常满意”或“非常满意（5分）”，否则最多重试 2 次。
- 单个字段重试失败后记录失败并继续下一个字段，不阻塞全局串行队列。
- 完成提示统一显示成功写入、跳过已有内容、重试后失败和未识别控件数量。
- 保持 SPA 会话锁、12 秒任务上限、15 秒 watchdog，以及不联网、不自动保存、不自动提交的安全边界。

## [1.0.0-beta.5] - 2026-07-06

- 增加全局单任务执行队列；填写任务支持取消、12 秒超时和最多 2 次重试。
- 增加 SPA 页面会话锁，路由变化时立即取消旧页面任务并清空待执行任务。
- 扫描前等待 DOM 连续稳定 800 毫秒，并额外等待 300 毫秒，避免过早扫描漏字段。
- Element UI 下拉框改为逐项串行处理，每项最长 1200 毫秒，只精确选择“非常满意”或“非常满意（5分）”。
- 文本框只填写空白项，填写前再次校验页面会话与当前值。
- 增加 15 秒安全看门狗，防止任务卡死。
- Queue 不再点击“保存”；每门预填后等待用户检查、手动保存并返回列表。

## [0.5.2] - 2026-07-03

- 修复 Queue 已进入详情页后仍被短时等待误判为进入失败的问题。
- 新增 `awaiting-detail`、`filling-detail`、`saving-detail`、`returning-list`、`awaiting-list` 可恢复状态。
- 详情页检测不再依赖 URL 变化、“设置分值”文字或文本框是否已完成渲染。
- 增加最长 15 秒的详情页稳定等待和详情诊断日志。
- 保持只保存课程草稿，不点击最终“提交评价”。

## [0.5.1] - 2026-07-03

- 修复 Queue 在 Element UI fixed-right 固定操作列中无法匹配“未填写”行与“评价”入口的问题。
- 主表格负责识别提交状态，固定右列按相同 `rowIndex` 配对操作入口。
- 增加列表页扫描诊断日志和表格结构变化提示。
- 保持只点击行内“评价”和详情页“保存”，不会点击最终“提交评价”。

## [0.5.0] - 2026-07-03

- 恢复双脚本结构。
- 新增 Lite 版：当前课程一键预填，不自动保存。
- 新增 Queue 版：列表页队列填写并自动保存当前课程草稿。
- 扩充随机评分和开放式评价模板池。
- 两个版本均不联网、不使用 AI、不收集数据。
- Lite 不自动保存、不自动提交。
- Queue 会自动点击详情页“保存”，但不会点击最终“提交评价”。
- 用户最终仍需人工检查并手动提交评价。

## [0.3.0] - 2026-07-03

### Changed

- 移除复杂面板，页面只保留右下角“教评辅助”按钮。
- 改为点击按钮后直接执行的一键极简预填。
- 增加课程优点和改进建议的本地随机模板组合。
- 增加少量“满意”、大部分“非常满意”的随机评分分布。
- 保持不联网、不自动保存、不自动提交。

## [0.2.0] - 2026-07-03

### Changed

- 移除 AI 版本，项目聚焦单一的本地版教评辅助工具。
- 脚本重命名为 `userscripts/suat-eval-helper.user.js`。
- 更新脚本名称、版本、下载地址和自动更新地址。
- 浮动按钮打开后先显示页面预检，不再立即填写。
- 支持修改优点评价和建议评价两段模板，并保存到浏览器本地存储。
- README 和教程改为单版本一键安装流程。

### Safety

- 保持完全离线，不收集或上传数据。
- 用户确认后才预填。
- 不覆盖已有评分或文字。
- 不自动保存、不自动提交、不绕过登录或验证码。

## [0.1.0] - 2026-07-03

### Added

- 新增本地用户脚本，支持识别 SIAT/SUAT 课程评价页。
- 新增评分下拉框辅助选择和开放式评价模板预填。
- 新增 SPA 页面切换支持。
- 明确禁止自动保存、提交、确认和批量评价。
