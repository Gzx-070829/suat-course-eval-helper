# AI 模式配置与隐私说明

AI 版通过用户自行选择的 **OpenAI-compatible Chat Completions API** 生成开放式评价草稿。AI 模式默认关闭；固定模板模式不会发起外部网络请求。

## 使用前提

你需要从所选 API 服务商获得以下三项信息：

- API Base URL
- Model（模型名称）
- API Key

不同服务商的地址、模型名、计费和数据保留政策不同，请以服务商官方文档为准。不要把 API Key 发到 Issue、聊天群或截图中。

## 如何填写

### API Base URL

填写 OpenAI-compatible API 的基础地址。例如服务商文档若给出：

```text
https://api.example.com/v1
```

就把它原样填入 Base URL。脚本会移除末尾多余的 `/`，并请求：

```text
POST https://api.example.com/v1/chat/completions
```

如果你直接填写了以 `/chat/completions` 结尾的完整地址，脚本不会重复追加路径。

请只使用可信的 HTTPS 服务地址。`@connect *` 是为了兼容用户自行选择的服务商，不代表脚本会在后台访问任意网站；网络请求只会在用户主动点击“生成草稿”后发生。

### Model

填写服务商支持 Chat Completions 的准确模型标识，例如服务商控制台或文档中列出的模型名。模型名不正确通常会收到 “model not found” 一类错误。

### API Key

填写服务商签发的密钥。点击“保存设置”后，脚本通过 `GM_setValue` 将它保存在 Tampermonkey 的本地脚本存储中，不会硬编码进 userscript。

本地保存不等于零风险：有权访问浏览器配置或 Tampermonkey 数据的人仍可能获得密钥。建议使用权限和额度受限的独立 Key，并在怀疑泄露时立即撤销。

## 请求格式

脚本向 `{baseUrl}/chat/completions` 发送以下结构：

```json
{
  "model": "用户填写的模型名",
  "messages": [
    { "role": "system", "content": "课程评价写作约束" },
    { "role": "user", "content": "课程上下文与用户补充" }
  ],
  "temperature": 0.7
}
```

API Key 通过 `Authorization: Bearer ...` 请求头发送给用户配置的服务。

## 完整流程

1. 在 AI 版面板中把“评价模式”从“固定模板”改为“AI 草稿”。
2. 选择语气，可填写真实、必要的用户补充。
3. 填写 Base URL、Model、API Key，并可点击“保存设置”。
4. 点击“生成草稿”。
5. 在确认框中查看将被发送的数据类型；只有确认后才会请求 API。
6. 在面板草稿框中阅读并修改结果。此时草稿尚未写入学校页面。
7. 确认草稿符合真实体验后，点击“预填页面”。
8. 再次逐项检查学校页面，最后由你手动保存或提交。

## 隐私风险

生成草稿时，以下信息会发送到你配置的 API 服务：

- 从当前页面提取的课程名称；
- 从当前页面提取的教师姓名；
- 页面上的评价问题文本；
- 你在“用户补充”中填写的内容；
- 所选语气和生成约束。

这些数据的处理、日志、存储地点、保留时间和可能的模型训练用途由 API 服务商决定。本项目不控制服务商，也不为其隐私实践背书。使用前应阅读服务商的隐私政策和数据处理条款。

请不要在“用户补充”中输入学号、身份证号、手机号、邮箱、登录密码、Cookie、Token、其他人的个人信息或与评价无关的敏感内容。若课程名或教师名也不适合发送给第三方，请使用固定模板模式。

## AI 输出边界

提示词要求模型生成 60–120 字、真实、礼貌、具体且建设性的中文草稿，并禁止夸张吹捧、编造事实或替用户作最终判断。但模型仍可能违反要求。用户必须预览、修改和确认；AI 草稿不会自动写入，更不会自动保存或提交。
