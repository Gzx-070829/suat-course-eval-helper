# AI 版使用与隐私说明

## AI 版适合谁

AI 版适合已经拥有 OpenAI-compatible API，并希望根据真实感受生成个性化草稿的用户。如果你不知道 API Key、Model 或 Base URL 是什么，请使用默认推荐的 Lite 版。Lite 不联网、无需配置，也能根据题目类型生成本地草稿。

## 首次设置

打开右下角“教评辅助 AI”，展开“首次设置 / 高级设置”。填写：

- **API Base URL**：API 服务的基础地址，例如 `https://api.example.com/v1`。
- **Model**：服务商提供的准确模型名称。
- **API Key**：服务商签发的密钥。

点击“保存设置”后，这些信息保存在 Tampermonkey 的本地脚本存储中。面板只用掩码提示 Key 已配置。可以随时点击“清除 API Key”。

脚本使用 OpenAI-compatible Chat Completions 请求：

```text
POST {Base URL}/chat/completions
Authorization: Bearer {API Key}
```

不同服务商的模型名称、价格、日志和数据保留规则不同，请以服务商官方说明为准。

## 使用流程

1. 输入自己的真实感受。
2. 选择语气。
3. 点击“生成 AI 草稿”。
4. 阅读将发送哪些信息的隐私提示。
5. 确认后等待生成。
6. 逐题阅读和修改可编辑草稿。
7. 点击“采用并预填”。
8. 检查学校页面，并由本人手动保存。

未配置 API 时，点击生成只会展开设置区域，不会联网。

## 什么时候会联网

只有同时满足以下条件时才会联网：

1. API 设置完整。
2. 用户主动点击“生成 AI 草稿”。
3. 用户确认隐私提示。

页面加载、输入真实感受、预览草稿、采用草稿和撤销时都不会请求 AI API。

## 会发送哪些数据

- 课程名称
- 教师姓名
- 页面中的开放题问题
- 用户输入的真实感受
- 用户选择的语气

这些信息发送到用户自己配置的 API 服务，而不是项目维护者服务器。

## 不会发送哪些数据

脚本不会主动发送学校密码、Cookie、Token、验证码、学号或学校登录凭据。请不要把这些敏感信息写进“用户真实感受”。

## 常见错误

- **API Key 无效**：Key 错误、过期或没有模型权限。
- **模型名称可能错误**：Model 与服务商实际名称不一致。
- **Base URL 无法访问**：地址错误、网络受限或服务不兼容。
- **请求超时**：服务响应过慢，可以稍后重试。
- **服务额度不足**：账户余额不足或触发频率限制。
- **返回格式异常**：服务没有按要求返回草稿数据，可以重试或更换兼容模型。

不要在 Issue、聊天群或截图中泄露 API Key。诊断信息不会包含完整 Key、用户草稿或课程评价内容。

## AI 输出边界

AI 输出只是写作草稿。模型没有亲自上过课程，可能产生空泛、不准确或用户并不认同的表达。所有草稿必须由用户预览、编辑和确认；脚本不会自动保存或提交。
