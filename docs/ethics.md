# 伦理原则与安全边界

## 为什么保留 Lite 版

Lite 只填写当前课程，不自动保存，适合希望逐门检查、完全掌握保存时机的用户。它是风险最低、最容易理解的默认版本。

## 为什么 Queue 也不自动保存

Queue 的自动化边界止于详情页预填。每门课程都由用户检查并手动保存，最终“提交评价”也必须由用户亲自决定。

Queue 只处理状态为“未填写”的行，默认跳过“未提交”和“已提交”，避免覆盖已经保存或提交的评价。

## 为什么不使用 AI

课程评价应来自学生本人的真实体验。自动生成看似完整的判断容易造成不准确和千篇一律，因此项目只使用可检查的本地随机模板。

## 隐私原则

两个版本都在浏览器本地运行，不联网、不收集或上传数据，也不绕过登录、验证码或权限控制。Queue 的运行进度只保存在当前浏览器中，并使用哈希标记课程，不明文记录课程名或教师名。

## 用户责任

用户应对最终评分和文字负责。随机内容只是编辑起点，必须逐项检查并按真实体验修改。

本工具只减少重复点击和重复输入，不替代真实评价，不应被用于虚假评价、代评或无人检查的批量最终提交。
