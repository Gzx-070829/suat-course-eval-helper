# 伦理原则与使用边界

## 为什么不接入 AI

课程评价应来自学生本人对课程的真实体验。自动生成看似完整的评价，容易带来不准确、过度推断和千篇一律的问题，也可能让用户在没有认真阅读的情况下直接采用。

因此，本项目只提供本地模板和重复操作辅助。模板是可编辑的表达起点，不是对课程和教师的自动判断。

## 为什么坚持人工检查

学校页面结构可能变化，题目含义也可能不同。即使脚本进行了预检和安全识别，仍可能出现控件未识别、模板不适合或页面状态变化。

用户必须逐项检查评分和文字，并根据真实体验修改。工具不能替代人的判断。

## 为什么坚持人工保存

保存或提交会让评价正式生效。自动执行最终操作会跳过用户的最后检查，也可能把错误内容直接提交。

因此，脚本不会查找或点击学校页面上的保存、提交、确认、完成或下一步按钮。最终操作始终由用户本人完成。

## 项目只减少重复操作

本项目可以减少重复选择和重复输入，但不能：

- 替代用户作出真实评价
- 鼓励虚假评价或代评
- 自动切换课程或批量提交
- 绕过登录、验证码或权限控制
- 收集或上传学校页面数据

如果学校规则禁止使用此类辅助工具，用户应遵守学校规则。
