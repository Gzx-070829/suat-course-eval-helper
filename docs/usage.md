# 使用教程

本文介绍如何安装和使用 Lite 版或 AI 版。建议先使用 Lite 版熟悉流程；同一时间只启用一个版本。

## 1. 安装篡改猴

1. 打开浏览器扩展商店或 [Tampermonkey 官网](https://www.tampermonkey.net/)。
2. 选择与你的浏览器对应的版本并安装。
3. 按浏览器提示授予扩展运行用户脚本所需的权限。
4. 确认浏览器工具栏中可以看到 Tampermonkey 图标。

请从可信来源安装扩展，并留意浏览器商店中显示的开发者信息。

## 2. 新建脚本

1. 点击浏览器工具栏中的 Tampermonkey 图标。
2. 选择“添加新脚本”或“Create a new script”。
3. Tampermonkey 会打开一个带示例内容的代码编辑器。
4. 全选并删除编辑器中的示例内容。

## 3. 粘贴 Lite 或 AI 脚本

选择一个版本：

- Lite：打开 `userscripts/suat-eval-helper-lite.user.js`，复制完整内容并粘贴到编辑器。
- AI：打开 `userscripts/suat-eval-helper-ai.user.js`，复制完整内容并粘贴到编辑器。

使用 `Ctrl+S`（macOS 为 `Command+S`）保存。回到 Tampermonkey 管理面板，确认脚本处于启用状态。

不要同时启用 Lite 和 AI 版；否则页面可能同时出现两个操作界面，造成重复预填。

## 4. 打开学校教评页面

正常登录以下任一学校教育信息化平台：

- `https://education.siat.ac.cn/`
- `https://education.suat-sz.edu.cn/`

按学校平台的正常入口进入某门课程的评价详情页。脚本不会也不能绕过登录、验证码或权限控制。

脚本通过以下信号识别评价页：网址路径包含 `/teaching/evaluation`，或者页面中出现“评价详情”“课程名称”“教师”等多个关键词。平台为 SPA 时，即使不刷新页面直接切换到评价详情，界面也会在短暂延迟后出现。

## 5. 点击辅助按钮

### Lite 版

1. 页面右下角出现“辅助填写教评”按钮后，点击它。
2. 阅读确认框。确认后脚本才会开始预填。
3. 脚本会依次滚动到 Element UI 下拉框，查找“非常满意”或“非常满意（5分）”。
4. 脚本会为仍为空白的开放式评价框填写默认模板。
5. 结束时查看统计信息，包括发现的下拉框、成功选择、失败/跳过和填写文本框数量。

### AI 版

1. 使用右下角面板选择评分：“非常满意”“满意”或“不自动选择”。
2. 固定模板为默认模式，不会访问外部 API。
3. 若选择 AI 草稿，填写 Base URL、Model 和 API Key，点击“保存设置”。
4. 填写可选的用户补充，选择语气，然后点击“生成草稿”。
5. 确认发送范围后等待生成。生成内容只会出现在面板草稿框里。
6. 阅读并修改草稿，确保它符合你的真实体验。
7. 点击“预填页面”，确认后脚本才会把内容写入学校页面。

AI 服务配置详见 [AI 模式](ai-mode.md)。

## 6. 检查后手动保存

预填结束不等于评价完成。请逐项完成以下检查：

1. 每个评分是否与自己的真实感受一致。
2. 脚本是否跳过了不属于评分项的下拉框。
3. 开放式文字是否回答了对应的问题。
4. 文字中是否有不准确、模板化、夸张或 AI 编造的信息。
5. 是否误含个人信息或不应提交的内容。

确认所有内容无误后，再由你本人手动点击学校页面上的保存或提交按钮。脚本永远不会替你完成这一步。

## 常见问题

### 页面没有出现按钮或面板

确认当前网址属于支持域名、脚本已启用，并已经进入评价详情页。可等待两秒，或在不丢失已填写内容的前提下刷新页面。若平台页面结构已经更新，请提交脱敏后的 Issue。

### 有些下拉框被跳过

脚本只会选择当前下拉菜单中能精确找到目标评分文字的项目。找不到目标、控件禁用或页面组件未正常打开时会跳过，以避免误选。请手动完成这些项目。

### 文本框没有被覆盖

这是预期行为。脚本默认保留已有文字，防止覆盖用户已经完成的内容。如需替换，请先自行备份和清空该文本框，再重新预填。

### 脚本会自动提交吗

不会。脚本不自动点击“保存”“提交”或“确认”，不自动翻页提交，也不提供批量提交功能。
